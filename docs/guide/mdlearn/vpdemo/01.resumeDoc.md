# 基于vitepress的个人简历

## 安装 vitepress

- [🚀](https://vitepress.dev/zh/guide/getting-started#installation)

1. 安装 vitepress

```bash
yarn add -D vitepress
```

2. 安装导向

```bash
yarn vitepress init
```

3. 启动

```bash
yarn docs:dev
```

或者

```bash
yarn vitepress dev docs
```

## 简历  📝✨

### 分析

- 首先，需要考虑**身份**。
  - 可能是在找工作的**应届毕业生**，或者是有一定工作经验要**跳槽**的人。
- 接下来，需求是什么？
  - 简历包含哪些部分，每个部分应该放什么信息。
  - 深层需求是希望简历看起来更**专业**，**符合招聘方的期望，提高通过筛选的机会**。

- 之后，分析需求
  - **根据不同岗位调整模块顺序和内容**，比如技术岗突出项目经验，设计岗强调作品集
  - 需要给出具体的例子，比如如何用数据展示成果，如何根据岗位描述调整关键词。
  - 可能他们的简历需要中英文版本，需要考虑排版和语言规范。

- 最后，要确**保简历结构清晰，能够轻松找到所需信息**。  

### 内容

- 简历内容应与目标岗位匹配，突出与岗位相关的核心技能和经验。
- 可以使用**关键词**来突出重点，如“创新”、“团队合作”等。
- 可以使用**图表**和**数据可视化**来展示项目成果和经验。

### 示例

🔥 一份标准的个人简历通常包含以下核心模块和信息，根据求职者经验与目标岗位的不同，模块顺序和内容可灵活调整：

#### **1. 个人信息(必填)**

- 基本信息： 姓名、联系方式（电话/邮箱）、求职意向（明确岗位名称）
- 可选补充：
  - 居住地
  - 个人网站/作品集链接（设计、技术岗建议附上）
  - 出生日期
- 例如  
   `张琳 | 📞 138-1234-5678 | 📧 <zhanglin@email.com>  
    📍 上海 | 🔗 作品集：www.zhanglin.design  
      求职意向：UI/UX设计师（3年经验）`

#### 2. **教育背景**

- **学历信息**：学校名称、学位（本科/硕士等）、专业、就读时间
- **补充亮点**：
  - GPA（若高于3.3/4.0或排名前20%）
  - 核心课程（仅应届生或与岗位强相关时列出）
  - 荣誉奖项（奖学金、竞赛获奖等）
- 示例

```text
上海交通大学 | 硕士 | 计算机科学与技术 | 2019.09–2022.06  

- GPA：3.8/4.0（专业前5%）  
- 相关课程：人机交互、数据结构、机器学习  
- 荣誉：国家奖学金（2021）
```

#### 3. **工作经历**

- **公司信息**:
  - 公司名称
  - 岗位名称
  - 在职时间（精确到月）

- 工作内容
  - （用“动词+成果+数据”描述）
  - 突出与目标岗位相关的核心职责
  - 分点陈述（每段经历3-5点，按重要性降序排列）
- 示例

```text
腾讯科技 | 高级产品经理 | 2020.07–2024.06  
- 负责微信支付商户端产品迭代，推动年交易额增长25%（从800亿至1000亿）  
- 协调10人跨部门团队，3个月内上线风控系统，降低欺诈率15%
```

#### 4. **项目经验**（技术/研发岗必填）

​- **项目名称**、角色（负责人/核心成员）、时间

- **​项目描述**：技术栈、目标、量化成果
- **​适用场景**：
  - 应届生无全职经历时替代“工作经历”
  - 社招人员展示与岗位相关的独立项目或跨部门协作项目

#### 5. **技能与证书**

- 技术类（编程语言、工具）
- 语言类（英语、其他语言）
- 证书类（PMP、CFA、行业认证等）
- ⚠️ ​注意：避免“精通Office”等基础技能，除非岗位明确要求

#### 6.**其他可选模块**

| 模块名称 | 适用场景 | 示例内容 |
| --- | --- | --- |
| 自我评价 | 转行/跨领域求职时快速突出优势 | “5年跨境电商运营经验，熟悉欧美市场用户增长策略” |
| 获奖荣誉 | 应届生/竞赛经历丰富者 | “全国大学生创业大赛金奖（2022）” |
| 附加信息 | 展示个人特质或与岗位相关的兴趣 | - 技术博客（日均UV 1000+）  - 开源社区贡献者 |  
|​志愿者经历| 公益组织经历、体现软实力 | “国际红十字会志愿者，累计服务200小时”|

#### 7. **附加信息**

- 个人介绍（简洁自我介绍，突出与目标岗位匹配的特质）
- 兴趣爱好（与目标岗位相关的爱好，如摄影、绘画等）
- 其他（如获奖情况、个人经历等）

### 模块顺序建议

- **根据不同求职者背景灵活调整优先级：**

| 求职者类型 | 推荐模块顺序 |
| --- | --- |
| 应届生 | 教育背景 → 实习经历 → 项目经验 → 技能 |
| 社招（同行业） | 工作经历 → 项目经验 → 技能 → 教育背景 |
| 转行/跨领域 | 技能/证书 → 相关项目经验 → 工作经历 |

### **避坑指南**  

• ❌ 写“期望薪资”（面试前不宜透露）  
• ❌ 放证件照（除非外企或岗位明确要求）  
• ❌ 使用“个人简历”作为标题（直接写姓名更专业）  
• ❌ 大段文字描述（用分点+关键词提高可读性）  

### **建议**

- ✅ 文件命名于格式：
  - ✅ 命名：姓名_岗位_年限_电话.pdf（例：王小明_高级产品经理_5年_13812345678.pdf）
  - ✅ 格式：PDF优先（避免格式错乱），字号11-12pt，页边距0.5-1英寸。
- ​✅ 关键词匹配：根据招聘JD（职位描述）调整技能和经历关键词。
- ✅ 量化成果：用数据（百分比、金额、用户量）代替模糊描述。
- ✅ 一页原则：应届生/初级岗位控制1页，资深人士不超过2页。
